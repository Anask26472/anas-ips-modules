# Contributing

Thanks for contributing to this IPS module.

## Rules for contributions
- Keep code readable and small.
- Prefer simple functions over heavy abstractions.
- Add comments only where they help understanding.
- Do not commit secrets, private keys, datasets with restricted licenses, or virtual environments.
- For security-sensitive changes, add a short note in the pull request explaining the impact.

## Basic workflow
1. Create a branch.
2. Run tests locally.
3. Keep commits focused.
4. Open a pull request with a clear summary.
